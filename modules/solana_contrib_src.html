estimateTransactionSize

  • estimateTransactionSize(txEnvelope: TransactionEnvelope): number
    • Defined in work/saber-common/saber-common/packages/solana-contrib/src/utils/printTXTable.ts:135

    Builds a transaction and estimates the size in bytes. This number is primrily to be used for checking to see if a transaction is too big and instructions need to be split. It may not be 100% accurate.

    This is used in expectTXTable and is useful for increasing efficiency in dapps that build large transactions.

    The max transaction size of a v1 Transaction in Solana is 1232 bytes. For info about Transaction v2: https://docs.solana.com/proposals/transactions-v2

    Returns 8888 if the transaction was too big. Returns 9999 if the transaction was unable to be built.

    Parameters

    • txEnvelope: TransactionEnvelope

    Returns number

printAccountOwners

  • printAccountOwners(connection: Connection, plainObj: object): Promise<void>
    • Defined in work/saber-common/saber-common/packages/solana-contrib/src/utils/printAccountOwners.ts:38

    A useful tool for debugging account structs. It gives a quick glance at addresses and owners. It also converts bignums into JS numbers.

    Types converted:

    • big numbers: converted to native numbers
    • addresses: format in base58, and prints the owner in parentheses if the account exists
    • plain objects: recursively converts

    HINT: This function is mainly useful for the browser. If you are writing Rust integration tests, use debugAccountOwners from chai-solana instead, so that you don't have to pass in connection.

    Usage:

    await printAccountOwners(connection, depositAccounts);
    // using void is recommend in dapps to avoid slowing down the user experience
    void printAccountOwners(connection, depositAccounts);

    Example output:

    tests/awesomeTest.spec.ts:583:29 {

      tokenProgram: 'TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA (BPFLoader2111111111111111111111111111111111)'
    }

    WARNING: This may break silently if web3 changes its api. This is only intended for debugging purposes only. But it should be safe to use in production.

printTXTable

  • printTXTable(tx: TransactionEnvelope, transactionLogs: string[], message: string): void
    • Defined in work/saber-common/saber-common/packages/solana-contrib/src/utils/printTXTable.ts:34

    Takes in a simulation result of a transaction and prints it in a cool table.

       ┌─────┬───┬───┬───┬───────────┬──────┬─────┬──────┬───┐
       │index│iso│mar│cum│ programId │quota │used │ left │CPI│
       ├─────┼───┼───┼───┼───────────┼──────┼─────┼──────┼───┤
       │  0  │298│281│464│'ATokenG..'│200000│24270│175730│ 1 │
       │  1  │298│ 74│538│'ATokenG..'│178730│21270│157460│ 1 │
       │  2  │298│ 74│612│'ATokenG..'│157460│27277│130183│ 1 │
       │  3  │298│ 42│686│'ATokenG..'│130183│21270│108913│ 1 │
       │  4  │338│265│951│'qExampL..'│108913│76289│ 32624│ 3 │
       └─────┴───┴───┴───┴───────────┴──────┴─────┴──────┴───┘
    • index: the array index of the instruction within the transaction
    • iso: the isolated size of the instruction (marginal cost of only the instruction)
    • mar: the marginal size cost of the instruction (when added to previous)
    • cum: the cumulative size of the instructions up until that instruction
    • quota/used/left: BPF instruction compute unit info
    • CPI: the maximum depth of CPI (current limit in Solana is 4)

    Safe for browser usage. Can be conveniently run with txEnvelope.simulateTable()

    Parameters

    • tx: TransactionEnvelope
    • transactionLogs: string[]
    • message: string

    Returns void
